Overview

OmniSafE is a practical end-to-end framework for automatically evaluating the safety of LLM-based embodied agents. Given an off-the-shelf embodied agent, OmniSafE constructs executable safety test cases, adapts the agent to simulation, translates its actions into robot-control commands, and detects safety violations during task execution.

The framework combines an agentic pipeline for scene generation with a lightweight action compilation mechanism. A multi-agent collaborative system builds diverse, simulation-ready safety scenes at scale, while the compiler grounds heterogeneous agent outputs into a unified execution interface. This enables plug-and-play testing across different embodied agent architectures and reveals safety failures across the full decision-to-execution pipeline.

OmniSafE Dataset Samples

OmniSafE-150 is a collection of automatically generated safety test cases produced by OmniSafE during embodied-agent evaluation. Each sample links one hazard category to a concrete instruction, an executable simulation scene, and safety assertions. The complete dataset is available at omni2026/Omni-dataset.

Agentic Scene Generation Pipeline

OmniSafE automatically turns a safety-oriented instruction into an executable simulation scene through a multi-agent collaborative system. This is the generative engine behind the OmniSafE-150 shown below.

Blueprint Agent

Interprets the task intent and sketches the required room type, zones, and coarse spatial layout.

Retrieval + Placement Agents

Retrieve suitable 3D assets and instantiate them with commonsense and physical constraints.

Review Agent

Checks whether the generated scene faithfully supports the intended hazard and execution requirements.

Safety Evaluation of Embodied Agents

We use OmniSafE to conduct automated end-to-end safety evaluation across representative LLM-based embodied agents. The results reveal failures that only emerge after high-level decisions are grounded into executable robot actions.

Finding with OMNISAFE

Embodied agent safety is jointly determined by both the planning architecture and the backbone LLM.

Changing the LLM alone does not fully determine safety; the embodied-agent architecture also shapes how hazards are recognized, refused, or executed.

Finding with OMNISAFE

Execution-level safety failures are decoupled from planning-level failures.

Agents may appear safe at the planning stage yet still fail after grounding into physical actions, showing the need for execution-level evaluation.
